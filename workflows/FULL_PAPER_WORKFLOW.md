## Master Workflow Prompt Sequence

The CRES system operates through an 8-step prompt chain. Each step activates the relevant skill modules and produces specific deliverables that feed into the next step.

### Step 1: INITIATE (Week 1–2)
```
Use SKILL 01_Ideation → Run gap_analysis_prompt → Run hypothesis_forge →
Run novelty_calculator → Output: Research Question Document
```

### Step 2: LITERATURE (Week 3–6)

```
Use SKILL 04_Data Intelligence → Run analysis_engine_prompt →
```

### Step 5: WRITING (Week 21–24)
```
Use SKILL 05_Writing Excellence → Run abstract_mastery →
Run introduction_craft → Run methods_precision →
```

### Adversarial Review Protocol

For each section, answer these questions as a hostile reviewer would:

1. **Fatal Flaw Check:** Is there any methodological issue that invalidates the conclusions?
2. **Alternative Explanation Check:** Could the results be explained by confounds or artifacts?
3. **Generalizability Check:** Are the claims appropriately scoped, or do they overreach?
4. **Reproducibility Check:** Could another researcher reproduce this study from the information provided?
5. **Statistical Check:** Are all statistical tests appropriate, properly reported, and correctly interpreted?
6. **Logical Check:** Does every claim follow logically from the evidence presented?
7. **Novelty Check:** Does the paper make a contribution beyond what is already known?
8. **Ethical Check:** Are there any ethical concerns or unaddressed biases?

## Decision Points & Go/No-Go Criteria

### Go/No-Go Gate 1: After IDEATION (Week 2)
| Criterion | Go | No-Go |
|-----------|-----|-------|
| Novelty | At least 2 novelty dimensions confirmed | Incremental at best |
| Feasibility | Data accessible, methods within capability | Cannot access data or execute methods |
| Significance | Addresses a meaningful gap | Trivial question |
| Team alignment | Advisor/team supports direction | Fundamental disagreement |

### Go/No-Go Gate 2: After LITERATURE (Week 6)
| Criterion | Go | No-Go |
|-----------|-----|-------|
| Literature gap confirmed | Gap still exists after thorough review | Gap already filled by recent publication |
| Positioning clear | Contribution is specific and defensible | Contribution is vague or indistinguishable |
| No blocking paper | No published study that makes yours redundant | Identical study already published |

### Go/No-Go Gate 3: After METHODOLOGY (Week 10)
| Criterion | Go | No-Go |
|-----------|-----|-------|
| Ethics approved | IRB/ethics committee approval obtained | Ethics approval denied or requires major revision |
| Pre-registered | Study pre-registered before data collection | Cannot pre-register (exploratory only) |
| Power adequate | Sample size achievable within timeline | Cannot recruit sufficient sample |
| Methods feasible | Pilot confirms protocol works | Pilot reveals fundamental design flaw |

### Go/No-Go Gate 4: After DATA (Week 20)
| Criterion | Go | No-Go |
|-----------|-----|-------|
| Hypothesis testable | Results address hypothesis (support or refute) | Data cannot answer research question |
| Results interpretable | Findings tell a coherent story | Results are contradictory or uninterpretable |
| Sufficient power | Achieved power meets threshold | Underpowered study |

## Common Bottlenecks & Solutions

| Bottleneck | Typical Delay | Solution |
|------------|---------------|----------|
| Ethics approval delays | 2–8 weeks | Submit early; prepare standard responses to common concerns |
| Low recruitment/participation | 4–12 weeks | Have backup recruitment strategies; consider online platforms |
| Data access restrictions | 4–16 weeks | Negotiate data access before starting; have backup datasets |
| Analysis complications | 1–4 weeks | Consult statistician early; pre-specify handling of issues |
| Writer's block | 1–3 weeks | Write out of order; start with easiest sections; use pomodoro technique |
| Co-author delays | 2–6 weeks | Set clear deadlines; build in buffer time; send reminders proactively |
| Reviewer #2 (after submission) | 2–6 months | Use reviewer_response_mastery; prepare for R&R cycle |
| Contradictory results | 1–4 weeks | Explore systematically; may be genuine finding; consult literature for precedents |

## Risk Mitigation Strategies

| Risk | Probability | Impact | Mitigation |
|------|-------------|--------|------------|
| Hypothesis not supported | Medium | High | Frame as contribution regardless; have exploratory analysis plan |
| Ethics delay | High | Medium | Submit early; have template responses prepared |
| Data loss | Low | Critical | 3-2-1 backup strategy (3 copies, 2 media types, 1 offsite) |
| Co-author conflict | Medium | High | Establish authorship agreement in Week 1; regular communication |
| Scooped by another group | Low | Critical | Preprint early; focus on unique contribution angles |
| Analysis reveals fatal flaw | Low | Critical | Build in checkpoints; have contingency research questions |
| Journal rejection | High | Medium | Prepare 3 ranked target journals; have revision-ready materials |
| Reproducibility concerns | Medium | High | Pre-register; archive all code and data; use version control |

## Collaboration Protocol

### For Multi-Author Papers

**Week 1:** Authorship agreement
- Define author order and contributions using CRediT taxonomy
- Establish communication channels and meeting frequency
- Agree on decision-making process (lead author decides? consensus?)

**Biweekly:** Progress updates
- Lead author circulates progress summary
- All co-authors provide feedback within 48 hours

**Week 10 (Methodology):** Co-author review #1
- All co-authors review and approve methodology
- Sign-off on pre-registration details

**Week 20 (Data):** Co-author review #2
- All co-authors review results and interpretation
- Agree on framing of findings

**Week 24 (Writing):** Co-author review #3
- All co-authors review complete draft
- Provide consolidated feedback (not line-by-line from 5 people)

**Week 28 (Submission):** Final approval
- All co-authors confirm approval of submission
- Signed authorship forms collected

### Co-Author Feedback Template
```
SECTION: [e.g., Introduction, Methods]
FEEDBACK TYPE: [Major concern / Minor suggestion / Typo]
SPECIFIC LOCATION: [Page X, Paragraph Y]
CURRENT TEXT: "[quote]"
SUGGESTED REVISION: "[revision]"
RATIONALE: [why this change improves the paper]
```

## Full Paper Quality Checklist Before Submission

### Scientific Rigor
- [ ] Research question is clearly stated
- [ ] Hypothesis is specific and falsifiable
- [ ] Methodology is appropriate and rigorous
- [ ] Sample size is justified (power analysis)
- [ ] All pre-registered analyses completed
- [ ] Results directly address each hypothesis
- [ ] Effect sizes reported for all key findings
- [ ] Confidence intervals reported
- [ ] Limitations are honestly acknowledged
- [ ] Claims are proportional to evidence

### Writing Quality
- [ ] Abstract is self-contained and accurate
- [ ] Introduction establishes clear motivation and gap
- [ ] Methods provide sufficient detail for replication
- [ ] Results are presented without interpretation
- [ ] Discussion interprets without overclaiming
- [ ] Conclusion is forward-looking and specific
- [ ] Writing is clear, concise, and jargon-appropriate
- [ ] Transitions between sections are smooth
- [ ] All figures and tables are referenced in text
- [ ] No orphan references or missing citations

### Reproducibility & Ethics
- [ ] Pre-registration DOI included
- [ ] Data availability statement included
- [ ] Code available in public repository
- [ ] Ethics approval documented
- [ ] Informed consent described
- [ ] Conflict of interest statement included
- [ ] Funding sources acknowledged
- [ ] Author contributions in CRediT format

### Formatting & Compliance
- [ ] Word count within journal limits
- [ ] Figure resolution meets requirements (300+ DPI)
- [ ] Reference format matches journal style
- [ ] All required sections present
- [ ] Supplementary materials complete
- [ ] Cover letter prepared
- [ ] All co-authors have approved submission
- [ ] No copyrighted material used without permission
- [ ] Data protection regulations followed (GDPR, HIPAA, etc.)
