## When Rapid Publication Is Appropriate vs. Risky

### ✅ APPROPRIATE for Rapid Publication

| Scenario | Why Rapid Works |
|----------|-----------------|
| **Existing dataset analysis** | Data already collected; eliminates 10-week data collection phase |
| **Methodological replication/extension** | Protocol already validated by prior work |
| **Secondary data analysis** | Public datasets (census, WHO, World Bank) are immediately available |
| **Time-sensitive findings** | Policy windows, public health emergencies, breaking developments |
| **Conference deadline** | Hard deadline creates natural compression |
| **Commentary/response papers** | Responds to existing literature; no new data needed |
| **Meta-analysis** | Uses published data; no primary collection needed |
| **Computational simulation** | Can run analyses on available hardware without human subjects |
| **Pre-registered analysis of open data** | Combines existing data with new analytical questions |
| **Short communication formats** | Some journals explicitly invite brief, focused papers |

### ⛔ RISKY for Rapid Publication

| Scenario | Why It's Risky |
|----------|----------------|
| **Novel primary data collection** | Recruitment delays are unpredictable and uncontrollable |
| **Complex multi-experiment studies** | Each experiment may reveal needed revisions to the next |
| **Highly controversial claims** | Rushed methodology will be exploited by critics |
| **Requires new instrument development** | Validation takes time; unvalidated measures undermine everything |
| **Ethics-sensitive research** | IRB review cannot be rushed; corners cut here are fatal |
| **Qualitative research** | Thematic analysis requires iterative deep engagement |
| **Longitudinal designs** | By definition require extended time |
| **Researcher's first publication** | Insufficient experience to compress timeline safely |
| **Requires multiple co-author approvals** | Coordination overhead exceeds compressed timeline |
| **Effect sizes expected to be small** | Underpowered rapid studies produce unreliable results |

### Decision Matrix

```
                    DATA READY?    METHOD PROVEN?    LOW ETHICS RISK?    QUICK APPROVE?
Scenario A:            ✓               ✓                 ✓                   ✓         → RAPID ✓
Scenario B:            ✓               ✓                 ✓                   ✗         → HYBRID
Scenario C:            ✓               ✗                 ✓                   ✓         → CAUTIOUS
Scenario D:            ✗               ✓                 ✓                   ✓         → DELAY
Scenario E:            ✗               ✗                 ✗                   ✗         → FULL WORKFLOW ✓
```

## Trade-offs of the Rapid Timeline

### What You Sacrifice

| Dimension | Full Workflow (28 weeks) | Rapid Workflow (30 days) | Impact of Loss |
|-----------|--------------------------|--------------------------|----------------|
| **Literature depth** | 60–90 papers, full synthesis | 20–30 papers, targeted extraction | May miss important counter-arguments or related work |
| **Methodological refinement** | Pilot testing, iterative design | Pragmatic design, existing protocols | Higher risk of methodological limitations |
| **Sample size** | Powered for small effects | Relies on existing large datasets or accepts lower power | May miss small but meaningful effects |
| **Exploratory analysis** | Room for unexpected findings | Focused on pre-specified analyses | May miss serendipitous discoveries |
| **Writing polish** | Multiple revision cycles | One intensive pass + quick fixes | Less elegant prose, possible unclear passages |
| **Co-author input** | Multiple review rounds | Single consolidated review | Less diverse perspectives on interpretation |
| **Recovery time** | Buffer weeks for setbacks | No buffer — one delay cascades | High stress; schedule fragility |

### What You Preserve (Quality Floor)

| Dimension | Minimum Standard | How Rapid Maintains It |
|-----------|------------------|----------------------|
| **Statistical rigor** | Pre-specified analysis plan | Pre-register on Day 8; adhere strictly |
| **Reproducibility** | All code and data archived | Archive continuously; automate with scripts |
| **Honest reporting** | Full results including null findings | Template enforces complete reporting |
| **Ethical compliance** | Ethics approval before data access | Use only pre-approved datasets |
| **Citation integrity** | All claims supported by cited evidence | Citation check in quality review |
| **Logical coherence** | Argument chain validated | Focused logic check on Day 22 |
| **Claims-evidence alignment** | No overclaiming | Adversarial claims audit on Day 23 |

## Quality Floor: Non-Negotiable Standards

The following standards **MUST** be met regardless of timeline pressure. If any cannot be met, the rapid track must be abandoned in favor of the full workflow.

### 🔴 HARD STOPS (Violating These Means Switching to Full Workflow)

1. **Pre-registration before analysis** — No exceptions. If you cannot pre-register, you are not doing confirmatory research.
2. **Ethics approval before human/animal data** — No shortcuts. Use only pre-approved datasets.
3. **Adequate statistical power** — Minimum power of 0.80 for primary analysis. If the dataset cannot support this, redesign or switch to the full workflow.
4. **Complete reporting of all conducted analyses** — Including null results, failed manipulations, and exploratory analyses clearly labeled as such.
5. **No p-hacking or HARKing** — Hypotheses stated before analysis; no data-driven hypothesis revision presented as confirmatory.
6. **Data and code availability** — Must be archived and publicly accessible by submission date.
7. **No fabricated or manipulated data/figures** — Obviously. But in a rush, figure manipulation errors can happen. Double-check every figure.
8. **All co-authors approve submission** — Cannot submit without explicit approval from every listed author.

### 🟡 WARNINGS (Proceed with Extra Caution)

1. **Less than 30 papers in reference list** — May indicate insufficient literature coverage.
2. **Single study with no replication** — Acknowledge this limitation explicitly.
3. **Convenience sample** — Discuss generalizability limitations.
4. **Effect size below medium** — Ensure interpretation is appropriately cautious.
5. **No pre-print posted** — Consider posting pre-print to establish priority.

## Day 1–3: Rapid Ideation

**Goal:** Select a research question that is high-impact, immediately feasible, and has data ready.

### Day 1: Gap Scan & Question Generation (8–10 hours)

**Morning (4 hours): Domain Mapping**
- List 5 research questions you are already thinking about
- For each, assess: Do I have access to data? Is the method proven? Is ethics approval in place?
- Rank by feasibility (data availability × method readiness × ethics status)

```
CRES PROMPT: Use gap_analysis_prompt (compressed mode)
"Given my expertise in [DOMAIN] and access to [DATASETS/RESOURCES],
identify the top 5 most impactful research questions I could address
within a 30-day timeline. For each, rate: feasibility (1-10),
impact potential (1-10), and time-to-completion estimate."
```

**Afternoon (4 hours): Rapid Novelty Check**
- For top 3 questions, search for existing publications (30 minutes per question)
- Use Google Scholar, PubMed, or domain-specific databases
- Confirm no identical study exists; identify closest related work

```
CRES PROMPT: Use novelty_calculator (rapid mode)
"Assess the novelty of this research question: [QUESTION].
Compare against these related studies: [LIST]. What is the
incremental contribution? Rate novelty on dimensions:
conceptual, methodological, empirical, theoretical."
```

**Evening (1–2 hours): Decision**
- Select primary research question
- Document rationale for selection

### Day 2: Hypothesis Forging (8–10 hours)

**Morning (4 hours): Hypothesis Development**
- Formulate 2–3 specific, testable hypotheses
- Define independent and dependent variables
- Specify expected direction and magnitude of effects

```
CRES PROMPT: Use hypothesis_forge
"Given the research question [QUESTION] and these key findings
from the literature [FINDINGS], formulate 2-3 testable hypotheses.
For each, specify: the predicted relationship, the mechanism,
the expected effect size range, and potential boundary conditions."
```

**Afternoon (4 hours): Feasibility Stress Test**
- For each hypothesis, verify: data available? analysis feasible? results interpretable regardless of outcome?
- Eliminate any hypothesis that cannot withstand a null result
- Select primary hypothesis and 1 backup

**Evening (1–2 hours): Documentation**
- Complete 1-page research question + hypothesis document
- Share with advisor/collaborator for rapid feedback (24-hour deadline)

### Day 3: Rapid Research Design (8–10 hours)

**Morning (4 hours): Design Blueprint**
- Select research design (must use existing data or proven method)
- Define analysis plan at a high level
- Identify potential confounds and how to address them

**Afternoon (4 hours): Pre-Registration Draft**
- Draft pre-registration on OSF or AsPredicted
- This forces specificity before any analysis begins

```
CRES PRE-REGISTRATION TEMPLATE (Rapid):
1. Study title
2. Research question
3. Hypotheses (with predicted direction)
4. Data source (and access confirmation)
5. Variables: IV, DV, covariates, exclusion criteria
6. Analysis plan: primary test, covariates, multiple comparison correction
7. Power analysis or sample size justification
8. Contingency plans for missing data, outliers, failed assumptions
```

**Evening (1–2 hours): Submit Pre-Registration**
- Submit and obtain time-stamped confirmation
- **MILESTONE:** Research question locked; pre-registered

### Day 1–3 Deliverables
- [ ] Selected research question with feasibility justification
- [ ] 2–3 testable hypotheses
- [ ] 1-page research question + hypothesis document
- [ ] Pre-registration submitted and time-stamped
- [ ] Advisor/collaborator feedback received (or pending with 24h deadline)

## Day 4–7: Targeted Literature Review

**Goal:** Build a focused, strategic literature foundation — not comprehensive, but covering all essential bases.

### Day 4: Strategic Search (8–10 hours)

**Morning (4 hours): Core Literature Identification**
- Identify 10–15 MUST-CITE papers (foundational works, closest related studies, methodological precedents)
- Use citation tracking: who cites these papers? What do they cite?
- Search strategy: 2–3 targeted database searches with precise keywords

```
CRES PROMPT: Use systematic_review_prompt (rapid mode)
"Conduct a targeted search for literature on [TOPIC]. Focus on:
1) The 10 most-cited papers from the last 5 years
2) The 5 most directly relevant recent studies
3) The 3 key methodological papers for my analytical approach
4) Any studies that contradict my expected findings"
```

**Afternoon (4 hours): Reading & Extraction**
- Skim-read 20–30 papers; deep-read 10–15 most important
- Extract: main finding, method, limitations, relevance to your study
- Organize in a structured literature tracking spreadsheet

**Evening (1–2 hours): Gap Confirmation**
- Confirm your identified gap still exists after reading
- Note any studies that come close to your question

### Day 5: Literature Synthesis (8–10 hours)

**Morning (4 hours): Thematic Organization**
- Group papers into 3–5 themes
- For each theme: what is established? what is debated? what is unknown?

**Afternoon (4 hours): Synthesis Writing**
- Write a 1,500–2,000 word literature synthesis (not a full review — this will become your Introduction)

```
CRES PROMPT: Use synthesis_engine (rapid mode)
"Synthesize these [N] papers into a coherent narrative that:
1) Establishes the importance of [DOMAIN]
2) Identifies the specific gap my study addresses
3) Positions my contribution relative to existing work
4) Anticipates and addresses potential counter-arguments"
```

**Evening (1–2 hours): Citation Network**
- Map which papers cite which; identify key bridges
- Ensure no critical paper is missing from your list

### Day 6: Positioning & Argument Construction (8–10 hours)

**Morning (4 hours): Contribution Statement**
- Write a clear, specific contribution statement: "This study contributes to [FIELD] by [SPECIFIC CONTRIBUTION], which addresses the gap of [IDENTIFIED GAP]."
- Test contribution statement against novelty_calculator

**Afternoon (4 hours): Argument Architecture**
- Build the argument chain: Why does this matter? → What's missing? → What I do → What I find → What it means
- Map each claim to its supporting evidence (from literature or your data)

**Evening (1–2 hours): Bibliography Assembly**
- Format all references in target journal style
- Aim for 25–40 references (sufficient for rapid track)

### Day 7: Literature Review Finalization (8–10 hours)

**Morning (4 hours): Integration**
- Combine synthesis, positioning, and argument into a coherent Introduction draft
- Ensure smooth transitions between paragraphs

**Afternoon (4 hours): Quality Check**
- Verify every claim is supported by a citation
- Check for missing counter-arguments
- Ensure your gap is clearly and specifically identified

**Evening (1–2 hours): Completion**
- **MILESTONE:** Introduction/literature review section complete
- Bibliography finalized

### Day 4–7 Deliverables
- [ ] 25–40 key references identified and read
- [ ] Literature tracking spreadsheet
- [ ] 1,500–2,000 word literature synthesis
- [ ] Contribution statement
- [ ] Argument architecture document
- [ ] Formatted bibliography
- [ ] Introduction draft

## Day 8–10: Streamlined Methodology

**Goal:** Finalize a pragmatic, rigorous methodology and complete any remaining pre-registration details.

### Day 8: Method Design (8–10 hours)

**Morning (4 hours): Detailed Method Specification**
- Specify exact variables with operational definitions
- Detail data source, inclusion/exclusion criteria, and sample
- Define all measures and their psychometric properties

```
CRES PROMPT: Use experimental_design_prompt (rapid mode)
"Design the methodology for [RESEARCH QUESTION] using [DATA SOURCE].
Specify: (1) sample characteristics and selection criteria,
(2) variable operationalization, (3) analysis plan with
statistical tests, (4) assumptions and how to verify them,
(5) contingency plans for assumption violations."
```

**Afternoon (4 hours): Analysis Pipeline Construction**
- Write analysis code skeleton (R, Python, or Stata)
- Include: data loading, cleaning, primary analysis, secondary analyses, sensitivity checks
- Use existing code packages/libraries wherever possible

**Evening (1–2 hours): Method Documentation**
- Draft Methods section (1,500–2,000 words)
- Include sufficient detail for replication

### Day 9: Validation & Assumption Checks (8–10 hours)

**Morning (4 hours): Pre-Analysis Checks**
- Verify data integrity: missingness patterns, variable distributions, outliers
- Test statistical assumptions: normality, homoscedasticity, multicollinearity
- Document any deviations from ideal conditions

**Afternoon (4 hours): Power Verification**
- Run confirmatory power analysis with actual sample characteristics
- If underpowered, consider: narrowing scope, focusing on larger effects, or acknowledging limitation

**Evening (1–2 hours): Code Testing**
- Test analysis pipeline with small subset of data
- Verify outputs are sensible
- Debug any issues

### Day 10: Method Finalization (8–10 hours)

**Morning (4 hours): Complete Pre-Registration**
- Update pre-registration with any refinements from Day 8–9
- Submit any amendments if needed

**Afternoon (4 hours): Methods Section Polish**
- Finalize Methods section
- Create analysis code documentation
- Prepare reproducibility materials

**Evening (1–2 hours): Go/No-Go Check**
- Verify: data is clean, code is tested, assumptions are met, power is adequate
- **MILESTONE:** Methodology locked; ready for analysis

### Day 8–10 Deliverables
- [ ] Complete Methods section draft
- [ ] Fully functional analysis code
- [ ] Data integrity and assumption check report
- [ ] Power verification
- [ ] Pre-registration finalized
- [ ] Reproducibility materials prepared

## Day 11–15: Data Analysis

**Goal:** Execute all analyses, produce results, and create publication-quality visualizations.

### Day 11: Primary Analysis (10–12 hours)

**Morning (5 hours): Execute Primary Analysis**
- Run pre-registered primary analysis
- Document all output precisely
- Calculate effect sizes and confidence intervals

```
CRES PROMPT: Use analysis_engine_prompt (rapid mode)
"Analyze [DATA] to test the hypothesis: [HYPOTHESIS].
Primary analysis: [SPECIFIED TEST].
Report: test statistic, df, p-value, effect size (Cohen's d / η² / r),
95% CI, and interpretation. Flag any assumption violations."
```

**Afternoon (5 hours): Results Interpretation**
- Whether hypothesis supported or not, interpret the finding
- Calculate practical significance metrics
- Prepare primary results table

**Evening (1–2 hours): Documentation**
- Save all output with clear labeling
- Begin results narrative

### Day 12: Secondary & Sensitivity Analyses (10–12 hours)

**Morning (5 hours): Secondary Analyses**
- Run all pre-registered secondary analyses
- Exploratory analyses (clearly labeled as such)
- Subgroup analyses if applicable

**Afternoon (5 hours): Sensitivity & Robustness**
- Test robustness of primary findings:
  - Alternative model specifications
  - Exclusion of outliers
  - Different covariate sets
  - Alternative operationalizations
- Document whether findings hold across specifications

**Evening (1–2 hours): Sensitivity Summary**
- Create robustness summary table
- Note any specifications where findings change

### Day 13: Visualization Creation (10–12 hours)

**Morning (5 hours): Primary Figures**
- Create 2–3 publication-quality main figures
- Each figure should tell one clear story
- Follow journal formatting guidelines

```
CRES PROMPT: Use visualization_master
"Create publication-quality visualizations for these results:
[RESULTS]. Requirements: (1) journal-standard format,
(2) color-blind friendly palette, (3) clear axis labels,
(4) self-contained (understandable without reading text),
(5) high-resolution (300+ DPI). Suggest figure types
that best communicate each finding."
```

**Afternoon (5 hours): Tables & Supplementary**
- Create formatted results tables (3–5 tables)
- Supplementary figures for sensitivity analyses
- Ensure all tables follow journal style

**Evening (1–2 hours): Figure/Table Quality Check**
- All figures readable at printed size
- All numbers in tables match analysis output
- Captions are complete and self-contained

### Day 14: Results Narrative (8–10 hours)

**Morning (4 hours): Results Section Draft**
- Write Results section (1,500–2,500 words)
- Present results in logical order: primary → secondary → sensitivity
- Integrate figures and tables with smooth narrative

**Afternoon (4 hours): Results-Discussion Bridge**
- Draft preliminary interpretation notes
- Identify key findings vs. secondary findings
- Note unexpected results that need discussion

**Evening (1–2 hours): Results Quality Check**
- Every statistic verified against analysis output
- Every figure and table referenced in text
- No results presented without appropriate statistical test

### Day 15: Analysis Finalization (8–10 hours)

**Morning (4 hours): Archive & Document**
- Archive all data, code, and output
- Push to GitHub/OSF with DOI
- Create analysis reproducibility script (one-click rerun)

**Afternoon (4 hours): Comprehensive Results Review**
- Review all results together: Do they tell a coherent story?
- Identify any remaining analyses needed
- Resolve any discrepancies between analyses

**Evening (1–2 hours): Completion**
- **MILESTONE:** All analyses complete; Results section drafted; data archived

### Day 11–15 Deliverables
- [ ] Primary analysis results (with effect sizes, CIs)
- [ ] Secondary analysis results
- [ ] Sensitivity analysis results
- [ ] Robustness summary table
- [ ] 2–3 publication-quality main figures
- [ ] 3–5 formatted results tables
- [ ] Supplementary materials (figures + tables)
- [ ] Results section draft
- [ ] Archived data and code with DOI

## Day 16–20: Writing Sprint

**Goal:** Produce a complete manuscript draft through parallel section writing.

### Writing Strategy: Parallel Section Approach

In the full 28-week workflow, you write sections sequentially. In rapid mode, you write sections in parallel across 5 days:

| Day | Morning (4h) | Afternoon (4h) | Evening (1–2h) |
|-----|-------------|-----------------|-----------------|
| 16 | Discussion: Key findings & implications | Discussion: Limitations & future directions | Review + revise |
| 17 | Methods: Final polish | Results: Final polish | Cross-check stats |
| 18 | Abstract | Conclusion | Title generation |
| 19 | Introduction: Final revision | Integration: Cross-section coherence | Transitions |
| 20 | Full read-through | Revision pass | **DRAFT COMPLETE** |

### Daily Word Targets

| Section | Target Word Count | Writing Day |
|---------|-------------------|-------------|
| Abstract | 200–300 | Day 18 |
| Introduction | 1,500–2,000 | Day 4–7 (drafted); Day 19 (revised) |
| Methods | 1,500–2,500 | Day 8–10 (drafted); Day 17 (revised) |
| Results | 1,500–2,500 | Day 14 (drafted); Day 17 (revised) |
| Discussion | 2,000–3,000 | Day 16 |
| Conclusion | 300–500 | Day 18 |
| **TOTAL** | **7,000–11,000** | |

### Day 16: Discussion Sprint (8–10 hours)

**Morning: Key Findings & Implications**
- Write Discussion opening: Summary of key findings (not repetition — interpretation)
- Write theoretical implications: How do findings advance understanding?
- Write practical implications: What should practitioners/policymakers do differently?

```
CRES PROMPT: Use discussion_depth (rapid mode)
"Given these results [RESULTS] and this literature context [CONTEXT],
write a Discussion section that: (1) interprets findings beyond
mere restatement, (2) connects to theory, (3) identifies practical
implications, (4) honestly addresses limitations, (5) proposes
specific future research directions."
```

**Afternoon: Limitations & Future Directions**
- Write limitations section: Be honest but not self-defeating
- Write future directions: Be specific and actionable
- Write closing paragraph: Take-home message

**Evening: Review**
- Read Discussion through for coherence
- Ensure Discussion addresses every result in Results
- Check for overclaiming

### Day 17: Methods & Results Polish (8–10 hours)

**Morning: Methods Polish**
- Add any missing details
- Verify every method claim is accurate
- Ensure reproducibility from Methods alone

**Afternoon: Results Polish**
- Verify every statistic against output files
- Ensure smooth transitions between results subsections
- Check all figures/tables are properly referenced

**Evening: Statistical Cross-Check**
- Independent verification of key statistics
- Confirm all p-values, effect sizes, and CIs are correct
- Check for internal consistency (e.g., df match sample size)

### Day 18: Abstract, Conclusion & Title (8–10 hours)

**Morning: Abstract**
- Write structured abstract: Background, Methods, Results, Conclusion
- Target 200–300 words (journal dependent)
- Most important 300 words in the paper — spend time here

```
CRES PROMPT: Use abstract_mastery (rapid mode)
"Write a structured abstract for this manuscript: [SUMMARY].
Include: (1) 1-sentence context, (2) 1-sentence gap/purpose,
(3) 2-3 sentences method, (4) 2-3 sentences key results
(with effect sizes), (5) 1-sentence conclusion/implication.
Maximum 250 words."
```

**Afternoon: Conclusion**
- Write conclusion: 300–500 words
- Clear take-home message
- Forward-looking final statement

**Evening: Title Generation**
- Generate 5–10 title candidates
- Test each for: accuracy, specificity, discoverability (keywords), memorability
- Select top 2; get quick feedback if possible

### Day 19: Integration & Coherence (8–10 hours)

**Morning: Introduction Final Revision**
- Revise Introduction based on completed Discussion
- Ensure Introduction sets up exactly what Discussion delivers
- Strengthen gap identification and positioning

**Afternoon: Cross-Section Coherence**
- Read entire manuscript start to finish
- Check argument chain: Intro → Methods → Results → Discussion
- Fix transitions between sections
- Ensure consistent terminology throughout

**Evening: Reference Integration**
- Verify all in-text citations have reference list entries
- Verify all reference list entries have in-text citations
- Format references in target journal style

### Day 20: Full Manuscript Review (8–10 hours)

**Morning: Complete Read-Through**
- Read entire manuscript without stopping to edit
- Note overall flow, coherence, and readability

**Afternoon: Revision Pass**
- Address all notes from morning read-through
- Focus on: clarity, concision, precision
- Eliminate redundancy; tighten prose

**Evening: Draft Completion**
- **MILESTONE:** Complete first draft ready for quality review
- Save version; begin quality review preparation

### Day 16–20 Deliverables
- [ ] Complete Discussion section
- [ ] Polished Methods and Results
- [ ] Abstract
- [ ] Conclusion
- [ ] Title candidates (5–10)
- [ ] Integrated, coherent manuscript
- [ ] All references formatted
- [ ] Complete first draft

## Day 21–23: Quality Review

**Goal:** Identify and fix critical issues through focused adversarial review. This is NOT a comprehensive review — it targets the highest-risk areas.

### Day 21: Methodology & Results Audit (10–12 hours)

**Morning (5 hours): Methodology Critique**
- Answer: Would a skeptical reviewer find the methodology adequate?
- Check: Is the design appropriate for the claims? Are confounds addressed?
- Check: Are statistical tests appropriate? Assumptions met? Multiple comparisons corrected?

```
CRES PROMPT: Use adversarial_peer_review (rapid mode)
"Review this methodology as a hostile Reviewer #2: [METHODS].
Identify: (1) the 3 most damaging criticisms, (2) the 3 most
likely questions from reviewers, (3) any fatal flaws.
Be ruthless. This paper will be rejected if you miss something."
```

**Afternoon (5 hours): Results Audit**
- Verify: Every claim in Results is supported by a statistical test
- Verify: Effect sizes are interpreted (not just significance)
- Verify: No p-hacking — all analyses were pre-registered or clearly labeled exploratory
- Check: Internal consistency of all numbers

**Evening (1–2 hours): Priority Issue List**
- Compile all identified issues into HIGH/MEDIUM/LOW priority list
- Focus on HIGH issues that could cause rejection

### Day 22: Logic & Claims Audit (10–12 hours)

**Morning (5 hours): Logic Validation**
- Trace argument chain from Introduction through Conclusion
- Identify any logical gaps, non-sequiturs, or circular reasoning
- Check: Does the Discussion overclaim relative to Results?

```
CRES PROMPT: Use logic_validator (rapid mode)
"Validate the logical chain in this manuscript: [MANUSCRIPT].
For each section, check: (1) Does each claim follow from
preceding evidence? (2) Are there logical gaps? (3) Are there
unsupported leaps? (4) Is the conclusion proportional to
the evidence? Flag any logical violations."
```

**Afternoon (5 hours): Claims-Evidence Alignment**
- List every claim in the paper
- Map each claim to its supporting evidence
- Flag claims that overreach their evidence base
- Revise overreaching claims

**Evening (1–2 hours): Revision Planning**
- Prioritize revisions: Which issues are most likely to cause rejection?
- Plan revision sprint for Day 26–28

### Day 23: Critical Fixes Only (10–12 hours)

**Morning (5 hours): Fix HIGH Priority Issues**
- Address every issue that could cause outright rejection
- This may include: adding analyses, rewriting problematic sections, adding limitations

**Afternoon (5 hours): Fix Selected MEDIUM Issues**
- Address medium-priority issues that are quick to fix
- Defer complex medium issues if they would compromise the timeline

**Evening (1–2 hours): Final Quality Check**
- Re-read critical sections
- Verify all fixes are complete and don't introduce new issues
- **MILESTONE:** Quality review complete; critical issues resolved

### Day 21–23 Deliverables
- [ ] Methodology critique report
- [ ] Results audit report
- [ ] Logic validation report
- [ ] Claims-evidence alignment table
- [ ] Priority issue list (HIGH/MEDIUM/LOW)
- [ ] All HIGH priority issues resolved
- [ ] Selected MEDIUM issues resolved

## Day 24–25: Journal Strategy

## Day 26–28: Revision Sprint

**Goal:** Address remaining quality issues and polish the manuscript to publication standard.

### Day 26: Structure & Flow (8–10 hours)

**Morning (4 hours): Structural Revision**
- Verify argument flow: Introduction → Methods → Results → Discussion → Conclusion
- Check: Does each section set up the next?
- Check: Are transitions smooth?
- Fix any structural issues identified in quality review

**Afternoon (4 hours): Section-Level Polish**
- Introduction: Strengthen opening hook; clarify gap; sharpen contribution statement
- Methods: Ensure complete reproducibility
- Results: Improve clarity of statistical reporting
- Discussion: Deepen interpretation; strengthen implications
- Conclusion: Ensure memorable take-home message

**Evening (1–2 hours): Terminology Check**
- Ensure consistent use of key terms throughout
- Define all acronyms at first use
- Check for jargon that might confuse reviewers

### Day 27: Precision & Clarity (8–10 hours)

**Morning (4 hours): Paragraph-Level Revision**
- Each paragraph has a clear topic sentence
- Each paragraph has a single main idea
- Transitions between paragraphs are logical
- No paragraph is longer than 200 words

**Afternoon (4 hours): Sentence-Level Revision**
- Eliminate passive voice where active is clearer
- Remove unnecessary hedging ("it could perhaps be argued that")
- Remove excessive qualification ("somewhat," "rather," "quite")
- Fix any ambiguous sentences

**Evening (1–2 hours): Word-Level Polish**
- Check for repeated words/phrases
- Eliminate wordiness
- Verify technical terms are used correctly

### Day 28: Final Polish (8–10 hours)

**Morning (4 hours): Formatting Compliance**
- Final format check against journal guidelines
- Verify: margins, font, spacing, line numbers, page numbers
- Verify: reference format, figure captions, table formatting
- Verify: word count compliance (abstract and main text)

**Afternoon (4 hours): Final Read-Through**
- Read entire manuscript aloud (catches errors eyes miss)
- Have one other person read it if possible
- Fix any remaining issues

**Evening (1–2 hours): Version Lock**
- Save final version with clear naming convention
- Generate PDF for final review
- **MILESTONE:** Manuscript ready for submission

- [ ] Structurally revised manuscript
- [ ] Section-level polished manuscript
- [ ] Paragraph and sentence-level revisions complete
- [ ] Formatting compliance verified
- [ ] Final read-through complete
- [ ] Submission-ready manuscript saved

## Rapid Publication Journal Targets

### Journals Known for Fast Turnaround

| Journal Category | Example Journals | Typical First Decision | Format |
|-----------------|------------------|----------------------|--------|
| **Rapid Communication** | PLOS ONE, Scientific Reports, Heliyon | 2–6 weeks | Short communication |
| **Letters/Short Reports** | Nature Communications Letters, PNAS Direct Submission | 3–8 weeks | Letter format |
| **Open Access Mega-Journals** | PLOS ONE, SAGE Open, BMJ Open | 4–8 weeks | Standard article |
| **Preprint-Friendly** | eLife, PLOS, MDPI journals | 2–6 weeks | Standard article |
| **Field-Specific Fast Track** | Varies by field — research your niche | 2–4 weeks | Short communication |

### Selection Strategy for Rapid Track

1. **Prioritize journals with < 4-week first decision times** — Otherwise, the speed advantage of rapid publication is lost
2. **Prefer journals that accept short communications** — These have shorter review times by design
3. **Consider open access mega-journals** — They often have faster turnaround with adequate rigor
4. **Avoid journals with > 3-month review times** — Even if prestigious, they negate the rapid timeline
5. **Check recent publication dates** — A journal that used to be fast may have slowed down

## Emergency Protocols

### When You Discover a Fatal Flaw Mid-Rapid-Track

A fatal flaw is any issue that fundamentally undermines the validity of your conclusions. Examples: data integrity problem, assumption violation that cannot be corrected, confound that invalidates the design.

#### Protocol F1: Fatal Flaw in Data (Day 11–15)

```
STEP 1: STOP all analysis immediately
STEP 2: Document the flaw precisely
STEP 3: Assess: Can it be fixed?
  → YES: Fix and resume (may extend timeline by 2-5 days)
  → NO: Go to STEP 4
STEP 4: Assess: Is there an alternative analysis that avoids the flaw?
  → YES: Redesign analysis and resume (may extend by 5-10 days)
  → NO: Go to STEP 5
STEP 5: Can the research question be reframed to accommodate the flaw as a limitation?
  → YES: Reframe and continue with major limitation acknowledged
  → NO: ABORT rapid track → switch to FULL WORKFLOW with new design
```

#### Protocol F2: Fatal Flaw in Methodology (Day 8–10)

```
STEP 1: STOP methodology finalization
STEP 2: Assess: Is the flaw in the original design or the implementation?
  → ORIGINAL DESIGN: Switch to backup research question/hypothesis
  → IMPLEMENTATION: Redesign implementation; extend timeline by 3-7 days
STEP 3: If switching to backup: Does backup meet rapid publication criteria?
  → YES: Restart from Day 8 with backup question
  → NO: ABORT rapid track → switch to FULL WORKFLOW
```

#### Protocol F3: Results Contradict Hypothesis (Day 11–15)

```
NOTE: This is NOT a fatal flaw — it's a finding.
STEP 1: Verify the analysis is correct (re-run, check code)
STEP 2: If verified, this is a legitimate result
STEP 3: Frame as: "Contrary to prediction, [finding]. This suggests [interpretation]."
STEP 4: Consider: Does this change the contribution claim?
  → Still contributes knowledge: Continue with revised framing
  → No longer contributes: Assess if alternative analysis supports modified claim
STEP 5: Never HARK (present post-hoc interpretation as if it were the original hypothesis)
```

#### Protocol F4: Scooped by Another Publication (Any Day)

```
STEP 1: Obtain the competing publication immediately
STEP 2: Compare in detail with your work
STEP 3: Assess overlap:
  → IDENTICAL: ABORT → consider extending into replication/extension study
  → PARTIAL OVERLAP: Identify what your study adds that theirs doesn't
  → COMPLEMENTARY: Emphasize differences; cite their work; continue
STEP 4: If continuing: Accelerate timeline further; post preprint immediately
STEP 5: If scooped on the core contribution: ABORT rapid track
```

### Emergency Timeline Extensions

| Emergency | Additional Days | Total Rapid Timeline |
|-----------|-----------------|---------------------|

## Tips for Maximizing Speed Without Sacrificing Rigor

### Process Optimization

1. **Automate everything possible** — Use scripts for data cleaning, analysis, table generation, figure creation, and formatting
2. **Use templates** — Pre-built LaTeX/Word templates for target journals; pre-written Methods boilerplate for standard procedures
3. **Write while analyzing** — Draft Methods and Results framework while analyses are running
4. **Parallel processing** — Literature review and methodology can overlap; figure creation and writing can overlap
5. **Time-box every task** — Use Pomodoro technique (25 min work / 5 min break); set hard limits on literature reading (skimming > deep reading in rapid mode)
6. **Use AI assistance strategically** — CRES prompts can accelerate literature synthesis, argument construction, and drafting. Always verify AI-generated content.

### Cognitive Strategies

7. **Write now, edit later** — Separate generation from evaluation. First draft = brain dump. Quality review = surgical precision.
8. **Start with the easiest section** — Build momentum with Methods or Results before tackling Discussion or Introduction
9. **Use placeholder text for hard sections** — Write "[INSERT INTERPRETATION HERE]" and move on. Come back with fresh eyes.
10. **Work in focused 4-hour blocks** — No email, no phone, no social media. Deep work only.
11. **Track daily progress** — Word count, tasks completed, issues discovered. Visibility into progress maintains motivation.
12. **Sleep** — 7+ hours. Sleep deprivation produces sloppy work that takes longer to fix than the extra hours gain you.

### Collaboration Shortcuts

13. **Co-author standing approval** — Get pre-approval for submission with the understanding that you'll send the final version 48 hours before submission
14. **Consolidated feedback** — Ask co-authors for 5 bullet points, not line-by-line edits. You're the lead author; you make the prose decisions.
15. **Shared document** — Use Google Docs or Overleaf so co-authors can see real-time progress
16. **Daily stand-up** — 10-minute check-in with co-authors during writing sprint (Day 16–20)

### Technical Accelerators

17. **Use R Markdown / Jupyter Notebooks** — Combine code, output, and narrative in one document
18. **Use git version control** — Track every change; easy rollback if needed
19. **Use reference managers** — Zotero, Mendeley, or Paperpile for one-click citation insertion
20. **Use Grammarly or similar** — Automated grammar and style checking saves revision time
21. **Use the CRES prompt chain** — Each prompt is designed to produce publication-quality output in minutes rather than hours

## Rapid vs. Full Workflow Comparison

| Dimension | Full Workflow (28 weeks) | Rapid Workflow (30 days) |
|-----------|--------------------------|--------------------------|
| **Timeline** | 196 days | 30 days |
| **Literature depth** | 60–90 papers | 25–40 papers |
| **Methodology development** | Full design, pilot, refine | Pragmatic design, existing protocols |
| **Data** | Primary collection possible | Must use existing/approved data |
| **Writing process** | Sequential, multiple revisions | Parallel, focused revisions |
| **Quality review** | 2-week adversarial review | 3-day focused audit |
| **Best for** | Novel primary research | Secondary analysis, replications, timely findings |
| **Risk level** | Low (well-buffered) | Medium-High (fragile timeline) |
| **Stress level** | Moderate | High |
| **Success rate (first submission)** | ~30–40% | ~20–30% (but faster iteration) |
| **Total effort hours** | ~600–800 hours | ~250–350 hours |
| **Hours per day** | ~4–5 hours | ~8–10 hours |

### Hybrid Approach

For many researchers, the optimal strategy is a **hybrid**:

1. Use the **Full Workflow** for your primary research program (one major paper at a time)
2. Use the **Rapid Workflow** for secondary analyses, replications, commentaries, and time-sensitive findings
3. Run 1 full-track paper + 1–2 rapid-track papers simultaneously
4. This maximizes publication output while maintaining quality across your portfolio
